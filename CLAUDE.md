# Sillytavern-Memory 项目补充约束

## 项目方法论

- 本项目以 MCP 平台为核心开发点，定位是记忆交互平台/接口。
- 凡是参考项目中已有成熟可借鉴方案，优先复用和适配，不做无必要的 0 到 1 自造。
- 后续涉及导入清洗、LLM 执行器、上下文拼接、prompt 组织、召回策略等模块时，先查参考项目和现有成熟实现("refs\Engram"、"参考脚本\index.js"、"refs\ST-Bionic-Memory-Ecology")；只有在现有方案无法覆盖目标或与 MCP / nocturne_memory 语义冲突时，才补最小必要自定义。

- 制定计划先复盘参考项目模块
- 参考目标项目做分段提示词，提示词角色定义直接照搬，数据处理核心做独立特化
- 落地代码后立刻复查对照，哪里自作主张，哪里还能抄
- 借鉴为主，适配为辅
- 不开发独立算法，只做整合。信息来源项目路径内参考目标>搜索工具github查询
